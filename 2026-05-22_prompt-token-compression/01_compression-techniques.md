# Prompt/Rule 토큰 압축 기법

## 질문

Cursor Rule(.mdc) 등 AI 시스템 프롬프트의 토큰을 줄이면서 의미를 100% 유지하려면?

## 답변

### 기법 1: 산문(prose) → 구조화 포맷

가장 큰 절감 효과. 문장형 설명을 YAML/테이블/리스트로 변환한다.

**Before (42 tokens)**
```
새로운 주제가 등장하면 프로젝트 루트에 디렉토리를 만들어야 합니다.
디렉토리 이름은 날짜와 주제명을 포함해야 하며, 날짜는 작업 시작일 기준입니다.
주제명은 영문 kebab-case로 작성합니다.
```

**After (18 tokens)**
```
dir: `{YYYY-MM-DD}_{topic-kebab-case}/` at project root
```

### 기법 2: 불용어(filler) 제거

AI는 "해야 합니다", "반드시", "~하도록 합니다" 등 정중한 표현을 제거해도 동일하게 이해한다.

| 제거 대상 | 예시 |
|-----------|------|
| 조사/어미 장식 | ~해야 합니다 → ~한다 |
| 중복 강조 | 반드시 꼭 → (하나만) |
| 설명적 도입부 | "이 규칙은 ~를 위한 것입니다" → 삭제 |
| 당연한 문맥 | "AI에게" "이 프로젝트에서" → 삭제 |

### 기법 3: 명령형(imperative) 사용

주어, 조동사를 제거하고 동사로 시작한다.

**Before**: "기술 관련 Q&A가 발생하면 마크다운 파일로 기록해야 합니다"
**After**: "기술 Q&A → md 파일로 기록"

### 기법 4: 약어/기호 활용

AI가 이해하는 범위에서 기호를 사용한다.

| 기호 | 의미 | 예시 |
|------|------|------|
| `→` | ~하면, ~일 때, 결과 | `새 주제 → 디렉토리 생성` |
| `×` | 금지 | `× force-push to main` |
| `✓` | 필수/허용 | `✓ npm ci` |
| `/` | 또는 | `SSH/GPG 서명` |
| `e.g.` | 예시 | `e.g. git-rebase-guide` |

### 기법 5: 예시로 규칙 대체

장황한 설명 대신 입출력 예시 하나가 더 정확하고 짧다.

**Before (규칙 설명 50 tokens)**
```
디렉토리 이름에는 날짜를 YYYY-MM-DD 형식으로 앞에 넣고,
그 뒤에 언더스코어를 넣고, 주제를 영문 kebab-case로 넣어야 합니다.
파일은 번호를 앞에 붙이고 내용 요약을 kebab-case로 씁니다.
```

**After (예시 15 tokens)**
```
구조:
  2026-05-22_git-branch-strategy/
    01_rebase-vs-merge.md
```

### 기법 6: 중복 규칙 병합

같은 패턴의 규칙을 하나로 통합한다.

**Before**
```
- 프로젝트명을 일반 용어로 치환한다
- 조직명을 일반 용어로 치환한다
- 서비스명을 일반 용어로 치환한다
- 개인명을 일반 용어로 치환한다
```

**After**
```
도메인 고유명사(프로젝트/조직/서비스/개인명) → term-mapping.json의 replacement로 치환
```

### 기법 7: 네거티브 규칙 최소화

"하지 마라" 목록보다 "해라" 목록이 짧고 혼란이 적다.

**Before**
```
× 프로젝트명 사용 금지
× 조직명 사용 금지
× 서버 URL 사용 금지
× 개인명 사용 금지
✓ 오픈소스명은 그대로 사용
✓ 기술 표준명은 그대로 사용
```

**After**
```
도메인 고유명사 → 치환 (term-mapping.json 참조)
범용 기술 용어(OSS명, 프로토콜) → 그대로 유지
```

## 압축률 기대치

| 기법 | 토큰 절감률 |
|------|------------|
| 산문 → 구조화 | 40~60% |
| 불용어 제거 | 15~25% |
| 명령형 전환 | 10~15% |
| 기호 활용 | 5~10% |
| 예시 대체 | 30~50% |
| 중복 병합 | 20~40% |

복합 적용 시 원본 대비 **50~70% 토큰 절감** 가능.

## 주의사항

- AI 혼란 방지: 한 규칙 안에서 포맷을 섞지 않는다 (YAML이면 전부 YAML)
- 과도한 약어 금지: AI가 모호하게 해석할 수 있는 자체 약어 사용 자제
- 핵심 키워드 보존: 동사(생성/치환/금지)와 대상(파일명/브랜치명)은 절대 축약하지 않는다

## 참고

- OpenAI tokenizer 기준 한글 1글자 ≈ 2~3 tokens, 영문 1단어 ≈ 1 token → 영문이 토큰 효율적
- Cursor Rule은 매 대화마다 주입되므로 토큰 절감 효과가 누적됨
